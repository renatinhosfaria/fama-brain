# Objetivo
Definir como o Reno deve usar contexto de empreendimentos/produtos sem virar catálogo de imóveis.

# Regra central
Produto entra depois de diagnóstico mínimo. O Reno não deve despejar lista de imóveis no início.

# Uso correto
Empreendimentos devem ser usados para:
- conectar necessidade a possibilidades;
- validar faixa de valor;
- explicar perfil de região/produto;
- apoiar convite para visita;
- preparar handoff mais qualificado.

# Dados desejados por empreendimento
- nome;
- bairro/zona;
- tipo;
- faixa de preço;
- status de obra;
- dormitórios;
- perfil ideal de cliente;
- diferenciais reais;
- condições comerciais relevantes;
- riscos/limitações;
- quando mencionar e quando evitar.

# Política de atualização
CRM/estoque real continua fonte de verdade. O Obsidian pode guardar síntese comercial durável, mas disponibilidade e preço devem ser validados no sistema operacional antes de qualquer afirmação.

# Status atual
Inventário prioritário ainda precisa ser preenchido pela operação com produtos reais e critérios comerciais.

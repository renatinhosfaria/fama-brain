# Objetivo
Definir playbooks rápidos para o Reno decidir e executar registros úteis no segundo cérebro sem virar burocracia.

# Playbook 1 — Registrar lead relevante
## Gatilho
Lead revelou necessidade, trava ou próximo passo com impacto comercial.

## Passos
1. Resumir o lead em uma frase.
2. Nomear a trava principal.
3. Definir o próximo passo.
4. Atualizar lead timeline.
5. Anexar interaction se o contato trouxe detalhe relevante.

## Regra de ouro
Se a retomada futura melhorar, registra. Se não melhorar, não registra.

# Playbook 2 — Registrar broker recorrente
## Gatilho
O mesmo corretor começou a afetar repetidamente a qualidade do handoff ou do avanço.

## Passos
1. Identificar o padrão recorrente.
2. Descrever o gargalo sem julgamento pessoal.
3. Traduzir isso em implicação operacional para o Reno.
4. Atualizar broker profile.
5. Anexar broker interaction se houve contato relevante.

# Playbook 3 — Registrar aprendizado operacional
## Gatilho
Experimento, falha instrutiva ou padrão novo com potencial de reuso.

## Passos
1. Descrever o contexto.
2. Registrar a observação central.
3. Interpretar com cautela.
4. Definir ação sugerida.
5. Criar journal.
6. Se amadurecer, promover depois para decision ou shared context.

# Playbook 4 — Promover aprendizado para decisão
## Gatilho
Uma hipótese deixou de ser só hipótese e virou critério estável.

## Passos
1. Verificar se vale além de um caso.
2. Verificar se muda comportamento futuro.
3. Verificar se deve ser seguido até nova revisão.
4. Se sim para os três pontos, registrar decision.

# Playbook 5 — Não registrar
## Gatilho
O evento parece importante no momento, mas não melhora decisão futura.

## Sinais de descarte
- é só microdetalhe de conversa;
- já está bem representado no CRM;
- é ruído transitório;
- não existe próxima ação nem impacto futuro claro;
- exigiria reler toda a conversa para fazer sentido.

## Ação
Não registrar no segundo cérebro.

# Playbook 6 — Escolher a camada certa
## Se for
- estado vivo do negócio -> CRM
- fato pequeno e durável -> memória persistente
- contexto curado e estruturado -> Obsidian

# Playbook 7 — Revisão semanal do segundo cérebro
## Objetivo
Evitar inflação de ruído e consolidar aprendizado.

## Checklist
- há journals que já deveriam virar decision?
- há perfis sem uso real?
- há timelines infladas com detalhe irrelevante?
- há padrões recorrentes ainda não consolidados?
- há contexto compartilhado faltando para temas repetidos?

# Regra final
O segundo cérebro do Reno deve ficar pequeno o bastante para ser útil e rico o bastante para melhorar decisão. Se crescer como arquivo morto, falhou.
